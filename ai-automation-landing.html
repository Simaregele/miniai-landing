Логистика Комплаенс E-commerce Аналитика

Система анализа инвойсов для логистической компании

Проблема

Заказчик работает с огромным количеством поставщиков различных товаров по всему миру. В процессе работы ему регулярно приходится заполнять документацию на основе инвойсов, которые присылают эти поставщики. Речь идет о сотнях тысяч документов с разной структурой — инвойсы могут быть в текстовом формате, PDF, изображениях, сканах и так далее. Часто данные в них размещены непоследовательно или неочевидно, что затрудняет восприятие даже для человека. На момент начала проекта в компании около 40 сотрудников занимались исключительно извлечением данных из этих документов.

Архитектура системы

  • Классификация по языкам: Язык документа существенно влияет на качество извлечения данных.
  • Многоуровневая структура: Первый уровень включает OCR и извлечение текстовых и графических данных для правильного расположения информации. Данные преобразуются в Markdown для структурирования.
  • Поиск похожих документов: Используется векторная база данных для подбора релевантного шаблона. Например, для инвойсов от известных фабрик используется эталонный образец.
  • Извлечение данных: Данные извлекаются LLM-агентами с использованием систем на основе Chain of Thought (COT).
  • Валидация через интернет: Система ищет страницу товара в интернете по извлеченным данным для валидации полей (артикул, наименование, цвет). Также извлекается изображение товара для точной таможенной классификации (например, "Jacket" может быть и курткой, и пиджаком).
  • Система пост-обработки: После обработки каждый документ анализируется, выделяются поля с низким качеством распознавания для последующего улучшения системы.

Итог

В результате внедрения системы компания значительно увеличила объём обрабатываемых документов и расширила базу клиентов. Рост производительности напрямую повлиял на увеличение дохода, а время обработки инвойсов сократилось на 40%.

Система проверки товаров на включение в санкционные списки

Проблема

Компания, организующая доставку товаров в Россию и Беларусь, сталкивается с необходимостью проверки товаров на соответствие санкционным спискам. Это сложный процесс: списки постоянно обновляются, представлены в неструктурированном виде, а неправильный код ТН ВЭД может привести к блокировке товара и заморозке средств.

Архитектура системы

  • Сбор данных: Веб-скрапинг для сбора санкционных списков и нормативных актов.
  • Обработка документов: OCR, распознавание таблиц, очистка и нормализация данных.
  • Классификация ТН ВЭД: Сопоставление товара с кодом на основе описания и страны.
  • Анализ с помощью LLM: Языковые модели анализируют описание товара, выявляют риски и исключения.
  • Формирование обходных маршрутов: Если товар подпадает под санкции, система предлагает альтернативные пути.
  • Валидация: Результат предлагается специалисту для окончательного утверждения.

Итог

Реализация системы позволила сократить время на обработку запроса, существенно снизить риски, повысить прозрачность решений и ускорить логистические процессы. Компания может оперативно реагировать на изменения в санкционном регулировании, обеспечивая безопасность поставок.

Оптимизация поиска в магазине с 40 тысячами продуктов

Задачи, которые решает

Улучшение системы поиска для интернет-магазина с 40 тысячами позиций, где пользователи испытывали трудности с нахождением нужных товаров, что негативно сказывалось на выручке.

Каким образом решаются задачи

Разработан скрипт для автоматического формирования ключевых фраз (5-7 на продукт), описывающих различные сценарии использования товара. Эти фразы загружаются в систему поиска магазина как дополнительный индекс, что улучшает точность и релевантность результатов.

Улучшение процессов у клиента

  • Точность поиска: Помогает пользователям быстрее находить нужные товары.
  • Скорость работы маркетинга: Автоматизация освобождает время сотрудников.
  • Удовлетворенность клиентов: Улучшение пользовательского опыта повышает лояльность.

Техническая реализация

Использовались две языковые модели: Mistral 7B для быстрой генерации ключевых слов и ChatGPT-4 для улучшения их качества. Это позволило обойти технические ограничения и сократить время на обработку запросов.

LLM Ассистент для анализа товарных предложений конкурента

Задачи, которые решает ассистент

Ассистент автоматически собирает информацию о новых продуктах конкурентов и обновлениях существующих, сводя всё в одну актуализированную таблицу. Это позволяет компании быстрее и точнее реагировать на изменения на рынке.

Каким образом решаются задачи

Система использует комбинацию веб-краулера для сбора данных с сайтов, алгоритмов для извлечения информации из PDF-файлов и LLM для структурирования и анализа собранной информации.

Улучшение процессов у клиента

  • Эффективность мониторинга: Сокращение времени на ручной сбор данных.
  • Принятие решений: Повышение точности и оперативности благодаря актуальным данным.
  • Маркетинг и продвижение: Оптимизация кампаний и ценообразования.

Модификации и возможности

Система может быть адаптирована для управления запасами, адаптации ценообразования, разработки продуктов, генерации контента для маркетинговых кампаний и стратегического планирования.
